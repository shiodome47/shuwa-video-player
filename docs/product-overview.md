# Product Overview — Shuwa Learning Platform

## ビジョン

> 「学びたい」と思った瞬間に、学べる状態にある

手話学習に必要なすべてのもの — 動画、リソース、メモ、進捗 — を一箇所に集め、
学習を始めるまでの摩擦をゼロにする、自分専用の学習ホームを作る。

---

## このアプリが解決する問題

### 現状の摩擦

| 状況 | 問題 |
|---|---|
| DVDで学習したい | ケースから出す → プレイヤーに繋ぐ → 起動 → 前回箇所を探す |
| 単語を調べたい | ブラウザで「あのサイトどこだっけ」を探す |
| YouTubeで学習したい | チャンネルを都度検索する |
| 前回の続きを見たい | どこまで見たか覚えていない |
| 参考資料を探したい | ブックマークの海を漁る |

これらの摩擦が積み重なり、「学ぼう」という意欲が実際の学習行動に繋がらない。

### このアプリが提供する状態

- アプリを開くだけで前回の続きから動画が再生できる
- よく使う学習リソースがカテゴリ別に整理されていて、1クリックで開ける
- 学習に必要なものがすべて1つの画面に集まっている
- 管理は UI から行え、JSON手編集やフォルダルールを覚える必要がない

---

## アプリの本質的価値

このアプリは **動画プレイヤー** でも **ブックマーク管理ツール** でもない。

**「手話学習のホーム画面」** である。

ここに来れば、学習に必要なものがすべてある。
ここを起点に、学習が始まる。

---

## ターゲットユーザー

自分自身（プライベート用途）。

- 手話を学習している
- 学習素材: ローカル動画（元DVD）、外部URL動画、YouTube
- 学習補助: 単語検索サイト、YouTube チャンネル、参考ページ
- 使用環境: Mac（Chrome系ブラウザ）、将来的にモバイルも使いたい

---

## 設計原則

### 1. 摩擦ゼロの学習開始
アプリを開いたら5秒以内に動画が再生できる状態にある。

### 2. UI から育てる
初期データは空でよい。コース・レッスン・リソースはすべて UI から追加・編集・削除できる。
JSON手編集・フォルダルール・CLI操作を必要としない。

### 3. 学習の邪魔をしない UI
動画に集中できる、落ち着いた・静かなデザイン。派手さ・通知・余計な情報は不要。

### 4. 「ここに来れば見つかる」状態
動画もリンクもメモも、このアプリを開けば見つかる。散らからない。

### 5. 将来の拡張に耐える設計
動画ソース・データ永続化・UIレイアウトのいずれも、将来の要件変化に対して局所的な変更で対応できる。

---

## 成功の定義（MVP）

- [ ] アプリを開いてから10秒以内に動画再生が始められる
- [ ] 前回の再生位置から自動的に再開できる
- [ ] コース・レッスン・動画ソースをUIから追加・編集・削除できる
- [ ] よく使うURLをカテゴリ付きで登録・表示・開くことができる
- [ ] ローカル動画と外部URL動画の両方が再生できる

---

## スコープ外（このアプリではやらないこと）

- 動画のダウンロード・変換・エンコード
- 他ユーザーとのシェア・コラボレーション
- SNS連携
- 広告・収益化
- 手話認識・AI評価

## 用語定義

| 用語 | 定義 |
|---|---|
| Course | 学習コース（例: 手話技能検定5級テキスト）|
| Section | コース内の章・セクション（例: 第1章 挨拶）|
| Lesson | 最小学習単位。1つの動画に対応（例: おはようございます）|
| VideoSource | レッスンに紐づく動画の参照情報（local / remote / youtube）|
| ResourceLink | 学習に使う外部URL |
| ResourceCategory | リソースのカテゴリ（例: 単語検索 / YouTube）|
| Bookmark | 動画内の特定タイムスタンプへのブックマーク |
| Note | レッスンに紐づくメモ（タイムスタンプ付き可）|
| PlaybackState | 再生位置・速度・音量の記憶 |
| Progress | レッスンの学習進捗（完了・視聴秒数）|
